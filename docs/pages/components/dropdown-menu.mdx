# Dropdown and MultiSelect

Single-select (**`Dropdown`** / **`DropdownMenu`**) and multi-select (**`MultiSelect`**) with anchored or modal presentation, optional search, and theming. **Anchored** mode uses a **light scrim** (`rgba(0,0,0,0.12)`), **no modal fade animation**, and a **2px** gap under the trigger—closer to [react-native-element-dropdown](https://github.com/hoaphantn7604/react-native-element-dropdown) default behavior. **`mode="modal"`** keeps the stronger dimmer and fade.

## Dropdown (`DropdownMenu` / `Dropdown`)

`Dropdown` is the same component as `DropdownMenu` — use whichever name fits your codebase.

| Prop | Type | Description |
|------|------|-------------|
| items | `DropdownMenuItem[]` | Options (`id`, `label`, optional `disabled`) |
| value | string | Controlled selected id |
| defaultValue | string | Uncontrolled initial id |
| onValueChange | `(id: string) => void` | Selection callback |
| onChange | `(id: string) => void` | Alias for `onValueChange` (both fire if set) |
| placeholder | string | Empty label (default `Select`) |
| disabled | boolean | Disables the trigger |
| mode | `anchored` / `modal` | List placement |
| dropdownPosition | `auto` / `top` / `bottom` | Anchored only |
| maxHeight | number | List max height (excludes search row) |
| menuMinWidth | number | Anchored: min menu width |
| modalMaxWidth | number | Modal: max panel width (default `360`) |
| search | boolean | Filter field |
| searchPlaceholder | string | Search placeholder |
| onChangeSearchText | `(text: string) => void` | Search text |
| filterItem | `(keyword, item) => boolean` | Custom filter |
| renderItem | `(item, ctx) => ReactNode` | Custom row |
| showSelectedCheckmark | boolean | Default row leading check (default `true`) |
| style | `ViewStyle` | Root wrapper (like `style` in element-dropdown) |
| containerStyle | `ViewStyle` | List container / card (like `containerStyle`) |
| showsVerticalScrollIndicator | boolean | Default `true` |
| onOpen / onClose | `() => void` | Menu lifecycle |
| searchInputProps | `TextInput` props | Extra search field props |
| unstyled | boolean | Raw structure |
| slots | object | Fine-grained style overrides |
| testID | string | Test ids with suffixes |

## MultiSelect

| Prop | Type | Description |
|------|------|-------------|
| items | `MultiSelectItem[]` | Same shape as dropdown items |
| value | `string[]` | Controlled ids |
| defaultValue | `string[]` | Uncontrolled initial ids |
| onValueChange / onChange | `(ids: string[]) => void` | Selection changes |
| maxSelections | number | Optional cap |
| … | … | Same `mode`, `search`, `style`, `containerStyle`, `slots`, etc. as dropdown |

Menu stays open while toggling; dismiss with backdrop.

## Slots (both)

`root`, `trigger`, `triggerText`, `backdrop`, `menuContainer`, `menu`, `menuItem`, `menuItemText`, `menuItemSelected`, `searchInput`.

## Install

```bash
npx my-ui-lib add dropdown-menu
```

Multi-select ships from the same core package; no extra install.
